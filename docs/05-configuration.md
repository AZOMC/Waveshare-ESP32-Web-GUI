# 05 — Configuration

Most settings can be changed live from the dashboard without touching any code. A few things require editing a file and re-uploading, or editing the firmware and reflashing.

---

## Config Files (in `data/`)

Edit these JSON files in any text editor before running the LittleFS upload, or change most settings live from the dashboard.

### `users.json` — Login Accounts

```json
[
  {"username": "admin",    "password": "admin123"},
  {"username": "user1",    "password": "pass1"}
]
```

Up to 10 accounts are supported. After editing, re-run the LittleFS Data Upload.

> Passwords are stored in plain text — suitable for a closed LAN deployment only.

### `config.json` — MQTT, RS485, and SD Settings

This file is auto-generated by the firmware when you save settings from the dashboard. You can also pre-fill it before uploading. The key fields are:

```json
{
  "deviceName": "esp32",
  "mqttMode": "cloud",
  "mqttProto": "wss",
  "mqttHost": "your-broker-host",
  "mqttPort": 443,
  "mqttPath": "/mqtt",
  "inputs":  ["esp32/di1", "esp32/di2", "...", "esp32/di8"],
  "outputs": ["esp32/do1", "esp32/do2", "...", "esp32/do8"],
  "inputPubMs":  [0, 0, 0, 0, 0, 0, 0, 0],
  "outputPubMs": [0, 0, 0, 0, 0, 0, 0, 0],
  "rs485": { "baud": 9600, "parity": "None", "stopBits": 1, "dataBits": 8, "addr": "1",
             "pollIntervalMs": 500, "publishInterval": 1000 },
  "sdLogInterval": 60,
  "sdLogEnabled": true,
  "sdLogRotateInterval": 3600
}
```

### `telegram.json` — Telegram Credentials

```json
{
  "botToken": "",
  "chatId": "",
  "enabled": false,
  "rejectThreshold": 10
}
```

Can be filled in via the `/telegram` page instead of editing this file directly.

### `do_rules.json` — DO Logic Rules

Auto-generated when you save rules from the IoT Config page. Do not edit manually — use the UI.

## Changing Wi-Fi Credentials

Wi-Fi credentials are hardcoded in the firmware. Open `ESP32_GUI.ino` and update:

```cpp
const char* ssid     = "YOUR_SSID";
const char* password = "YOUR_PASSWORD";
```

Then re-flash the firmware. If Ethernet is always connected, Wi-Fi is only used as a fallback.

---

## Assigning a Fixed IP Address

The ESP32 uses DHCP by default. To keep the same IP after reboots, configure a **static DHCP lease** on your router — bind the board's MAC address to a fixed IP. The MAC address is printed in the Serial Monitor at boot.

## Changing Machine State Logic

By default:
- DI1 goes active → toggles RUNNING/STOPPED
- DI2 goes active → forces RUNNING
- DI3 goes active → forces STOPPED

To change this, find `autoUpdateMachineState()` in `ESP32_GUI.ino`. Channel indices are 0-based (DI1 = 0, DI2 = 1, ..., DI8 = 7):

```cpp
void autoUpdateMachineState(int ch, int state) {
  if (ch == 0 && state == 1) { setMachineState(1); return; }  // DI1 ON → RUNNING
  if (ch == 0 && state == 0) { setMachineState(2); return; }  // DI1 OFF → STOPPED
}
```

Re-flash the firmware after changes.

## Changing the Cycle Time Channels

Cycle time is currently measured from the **DI4** rising edge to the **DI6** rising edge. To use different channels, find this block in `ESP32_GUI.ino` (inside the DI scan in `loop()`):

```cpp
if (i == 3) { cycleStartMs = now; cycleActive = true; }          // DI4 — change index 3
if (i == 5 && cycleActive) { lastCycleTimeMs = now - cycleStartMs; cycleActive = false; }  // DI6 — change index 5
```

Re-flash firmware after changes.

---

## Changing the Reject Counter Channel

The reject counter currently increments on **DI7** (index 6). To use a different channel:

```cpp
if (i == 6) {        // ← change 6 to the desired channel index (0 = DI1)
  rejectCount++;
  ...
}
```

Re-flash firmware after changes.

---

## Changing the NTP Timezone

The board defaults to UTC+8 (Malaysia). Find this line in `ESP32_GUI.ino`:

```cpp
configTime(8*3600, 0, "pool.ntp.org", "time.cloudflare.com");
//          ↑ change this to your UTC offset in seconds
```

Re-flash firmware after changes.

---

## Increasing the Modbus Register Limit

The current maximum is 16 registers. To increase it:

```cpp
#define MAX_RS485_REGS 32   // was 16
```

Also increase the `StaticJsonDocument` sizes in `loadRS485Regs()` and `saveRS485Regs()`:

```cpp
StaticJsonDocument<8192> doc;  // was 4096
```

Monitor free heap on the Dashboard health panel after the change — each register uses ~64 bytes of RAM.
